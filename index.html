Prerequisites

  • You have a basic idea of what CAD software does. You might have used some type of CAD software, are an expert, or not.
  • You can use a computer.
  • You are inspired to create something.

What is OpenSCAD?

  • Open Source Computer Aided Design
  • Write scripts to design imaginary things. Make them in real life.
  • Easier than programming
  • We discuss other CAD software in SSD's Learn How to 3D Print workshop.
  • We can use OpenSCAD to create things to make with 3D printers.

Download and Install

  • Windows(tm), OS X(tm), Linux all supported with pre-compiled binaries.
  • Download from http://www.openscad.org/downloads.html.
  • Installation with pre-compiled binaries is straightforward. Double-click the downloaded installation file and install on your PC.
  • Use whatever version is tagged as the latest stable release.
  • If you need unreleased or specialized features you may need a development-branch binary or you may have to compile it yourself.

Let's SCAD

  • We create a text file with .scad extension = OpenSCAD file
  • Open the new file with OpenSCAD
  • cylinder(r = 50, h = 3);
  • Press F5 to compile
  • renders a cylinder with radius 50mm, height 3mm
  • cube([70, 70, 3]);
  • Press F5
  • renders a cube 70mm square, 3mm thick

Variables

  • Still in our new file, at the top.
  • » radius = 50;
  • » size = 70;
  • » thickness = 3;
  • » cube([size, size, thickness]);
  • » cylinder(r = radius, h = thickness);

What Did We Just Do?

  • In 4 lines of code we've created two CAD objects, a cube and a cylinder.
  • We introduced variables - also called parameters in CAD lingua.
  • We can adjust the size of the cube and cylinder by changing the values of size and radius.
  • We've written a program! We can go home now.

Get Modular

  • Save our new file.
  • Line break between our variables at the top, and code below it
  • While we're at it, » size = 2 * radius * cos(45);
  • Wrap our cube and cylinder with:
  • module WhatsIt() { ... }; where ... are the cube and cylinder declarations
  • press F5 -- what?!
  • near the top type:
  • WhatsIt();
  • press F5 - OK!

Centering

  • Center obj's with center = true param. Default is false.
  • Make a copy of the WhatsIt module -- change the name to WhatsItCentered
  • » cube([size, size, thickness], center=true);
  • » cylinder(r = radius, h = thickness, center = true);
  • Centers both objects
  • Cylinder is already centered on X, Y, so center attribute affects Z
  • call WhatsItCentered() from the top of our SCAD; instead of WhatsIt()
  • press F5
  • Objects can also be centered with the translate statement
  • Works with most objects
  • You can define a center attribute in your own module = OpenScad201

Positioning

  • We position objects using the translate statement.
  • translate([x-offset, y-offset, z-offset]);
  • Edit WhatsIt()
  • » translate([0, 0, -thickness / 2]) cylinder(r = radius, h = thickness * 2);
  • » translate([-size / 2, -size / 2, -thickness / 2]) cube([size, size, thickness]);
  • add the call to WhatsIt() again - at the top of our script
  • press F5

What Did We Just Do?

  • We used the translate statement to manually center each object.
  • Each object has its own translate statement. That means each object is positioned (centered) independentely of other objects.
  • » translate([size, size, 0]) WhatsIt();
  • uncomment/call WhatsItCentered();
  • Press F5
  • We see two copies of WhatsIt.

Rotation

  • We can rotate obects in OpenSCAD
  • rotate([x-rot, y-rot, z-rot]);
  • Make sure our OpenSCAD view has Axes enabled
  • » rotate([90, 0, 0]) WhatsItCentered();
  • Our module is rotated 90-degress on the X-axis
  • rotation values are in degress ranging 0 - 359

Nesting

  • Translate/rotate an object, module; or a set of declarations with {} nesting operators
  • {} groups declarations that we want to position or rotate
  • delete rotate([90, 0, 0]) from code, i.e. » WhatsItCentered();
  • In WhatsItCentered:
  • wrap our cube and cylinder with » rotate([90, 0, 0]) { ... }; where ... are the cube and cylinder statements
  • Press F5 - same result as before
  • statements like translate, rotate usually apply to the next (following) statement
  • braces are optional but always recommended even for single statements
  • translate/rotate are not the only modifiers that can use braces

Some Cleanup

  • Let's clean up our SCAD by deleting WhatsItCentered() - the module and call to it.
  • Also let's delete any "thickness * 2" code introduced in examples.
  • let's make something printable!

Include Libraries

  • Now let's grab a library for generating fonts.
  • http://www.thingiverse.com/thing:16193
  • Per instructions, place Write.scad and Letters.DXF in the same folder as our SCAD file.
  • » include <Write.scad> at the top of our file
  • add module SSD() { write("SSD", t = 5, h = font_size, center = true); }
  • call it
  • translate([0, 0, thickness * 2 - o]) SSD();
  • libraries must be in the same folder as our project
  • there is a global place for libraries in OpenSCAD installation directory
  • Check out MCAD - a parts library

Final Cleanup

  • We have two module calls: WhatsIt() and SSD()
  • Rename WhatsIt » EmblemBase(). Rename the call to it.
  • Wrap our calls to EmblemBase() and SSD() in a new module.
  • » module SSDEmblem() { union() { EmblemBase(); SSD(); } }
  • Call SSD_Emblem() at the top of our file.
  • Save our file.
  • We can now re-use the SSD embled as a module in other files if we want.
  • We just performed a basic exercise in additive/subtractive modeling. We have one object comprised of three smaller objects.
  • In the SCAD source that accompanies this presentation, we also added a new variable "o". This is used to prevent manifold errors in the output STL. Notice how we use this on the z-axis.

Export STL

  • Save our file again, just in case.
  • First compile with F6
  • Go to File » Export » Export as STL
  • Save exported STL file.
  • our STL file should be ready to convert to GCode for 3D printing
